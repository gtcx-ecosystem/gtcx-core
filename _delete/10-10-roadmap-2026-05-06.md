# 10/10 Roadmap — 2026-05-06

Move `gtcx-core` from the remediated `8.x/10` state to a defensible `10/10` by closing the remaining gaps in trust-path correctness, resilience, governance evidence, and operational verification.

## Completed

1. Traced certificate generation now uses real cryptographic signing.
2. Traced certificate verification now rejects unsigned and placeholder-signed artifacts.
3. Token signature assembly and verification now share one encoding contract.
4. Offline secure-storage lockout duration is now enforced correctly.
5. Verification templates and public input types now agree for `photo`, `custody-transfer`, and `settlement`.
6. Integration regression tests now cover repaired public security and verification behavior.
7. Offline queue replay now uses logical sequence ordering instead of wall-clock timestamps, with regression coverage for backward-clock behavior.
8. Risk-tier governance, agent evidence templates, and high-risk task playbooks now exist for reproducible agent work.
9. Enterprise supportability, migration, release artifact, and downstream-readiness docs now define the adoption contract explicitly.

## Phase 5 — Verification Reliability

Goal: make every standalone verification workflow observe current source behavior, not stale build artifacts.

Tasks:

1. Make integration tests resolve live workspace source during direct execution.
2. Re-run standalone integration tests to confirm no hidden dependency on prior builds.
3. Keep the root `pnpm test` path green after the resolver change.

Exit criteria:

1. `pnpm --filter @gtcx/integration-tests test` passes immediately after source edits.
2. No trust-path integration test depends on prebuilt `dist/` contents.

5. Run `pnpm typecheck`.
6. Run `pnpm build`.
7. Run `pnpm api:check`.
8. Run `pnpm security:threat-matrix`.

2. Any failure is either fixed or captured as an explicit residual blocker.
