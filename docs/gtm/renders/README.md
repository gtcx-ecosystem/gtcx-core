# Send-Ready Email Renders

Pre-substituted, copy-paste-ready renders of the [canonical sandbox intro template](../sandbox-intro-email-template.md) — one per target country, with every `{{placeholder}}` resolved from the template's parameter table.

The intent is to remove the last drafting step from the engagement critical path: when send-blocking gates (trust portal live, npm packages published, recipient verified, internal approval) clear, the email body below is ready to drop into a mail client.

| Render                                   | Country  | Status               |
| ---------------------------------------- | -------- | -------------------- |
| [`zimbabwe-2026.md`](./zimbabwe-2026.md) | Zimbabwe | Draft — send-blocked |

**Audience:** engagement leads at the point of send. The operational state (event log, decisions, blockers) for each engagement lives in [`docs/agile/engagement-log/`](../../agile/engagement-log/) — these renders are pure copy-paste artifacts.

## How to add a new render

1. Add the country row to the canonical [template parameter table](../sandbox-intro-email-template.md) if not present.
2. Substitute every `{{placeholder}}` with the country's row.
3. Save as `{country}-{year}.md` here with the same frontmatter shape as `zimbabwe-2026.md`.
4. Add to the table above.
5. Reference from the country's engagement log under `docs/agile/engagement-log/`.

Updating shared text (e.g., dropping "publish in flight" language once npm publish lands) is a one-line edit in the canonical template; each render is regenerated by re-running substitution.
